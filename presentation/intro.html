Motivation


  • Basic control techniques often do not take into account the myriad of constraints that the robot imposes.

  • They sometimes work, but in complex situations, they do not. However, they serve as a basis for more advanced controllers!

  • By "complex situations," we mean environments filled with obstacles, which may even be dynamic!

  • These situations are becoming increasingly common as robots move beyond the structured environment of factory floors and into our homes, for example.

Motivation


  • Examples of constraints:

    • Collision avoidance with obstacles/inter-agent;

    • Auto collision avoidance;

    • Joint limits in manipulators;

    • Input limits (velocity/torque)

    • Equilibrium constraints for humanoids.

Scope



  • We will learn techniques to model these constraints and control techniques to handle them.

  • The goal is to solve a task (e.g., achieving a pose) while abiding by the constraints.

  • The focus will be on manipulators. Among the types of constraints considered in this class, manipulators present the most complex case. Thus, learning how to handle them with manipulators also implies learning how to handle them in other types of robots.

  • Control-wise, the focus will be on 1st order kinematic control (i.e., velocity), although I will (briefly) discuss how can they be adapted to high order control and also consider dynamics.

Overview



  • This class is divided into three parts:

  • Part 1: we review some important concepts in robotics, as the concept of homogeneous transformation matrices, configuration, forward / inverse / differential kinematics, Jacobians, task functions and kinematic control.

Overview

  • Part 2: many of the constraints are related to distance computation. In the second part, a brief review of important concepts in analytic geometry will be given. We will then learn how to model the geometry of the robot and the environment. We will also discuss algorithms to compute distance and the associated quantities (e.g., Jacobians).


  • Ways to model geometry: convex shapes (CS), point clouds (PC), meshes;
  • Modeling robots using CS;
  • Review: convexity, point-to-object distance, projection, support function;
  • Overlap oracles, metrics and metric gradient;
  • CS-to-CS distance: GJK algorithm;
  • CS-to-CS distance: Von Neumann's alternating projection;
  • CS-to-PD distance: Bounding Volume Hierarchy (BVH);
  • Gradient of the distance function;
  • CS-to-CS smooth distance: modified Von Neumann's algorithm;
  • CS-to-PS smooth distance: ISI algorithm;
  • Signed distances;
  • Metric through neural networks;



Overview

  • Part 3: focus on how to formulate the controllers to achieve the task while considering the constraints. A brief review of optimization concepts will be given. Then, we will formulate the controller as an optimization problem (mainly a convex quadratic problem). A good emphasis will be given in the concept of Control Barrier Functions (CBFs).


  • Control vs Path Planning;

  • Path Tracking: The Vector Field Algorithm;

  • Control Barrier Functions;

  • Modeling constraints;

  • Review: convex optimization;

  • Formulating the control problem as an optimization problem;

  • Other approaches: VO, RVO, ORCA;

  • Reinforcement learning;




UAIBot

  • All the animations you saw so far were done using the UAIBot simulator.

  • Online robotics simulator (can be used in Google Colab or Jupyter Notebook) created by me and my students.

  • Built in javascript (3D visualization using threejs), Python (scripting) and C++ (heavy calculations). You will only have to code in Python, though.

  • We will use it in this class. It has (almost) all the algorithms that we discuss here implemented!

  • An internet connection is required!

UAIBot

  • There are many ways to use/install UAIBot;
  • It is first advisable to create a new Python environment. Then, (supposing you have .pip installed) type in the terminal:
    • 
>>pip install uaibot
  • After the installation, try running a simulation that is already implemented. For this, try the following code:
    • 
import uaibot as ub
    
sim = ub.Demo.constrained_control_demo_1()
my_path = '/home/vinicius/Desktop/my_uaibot_simulation'
#Change the path in 'my_path' accordingly
sim.save(my_path,'constrained_control_demo_1')
    and then go to the folder (/home/vinicius/Desktop/my_uaibot_simulation in this case) and open the HTML file to see the simulation.
